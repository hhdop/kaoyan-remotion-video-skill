# 考研类视频生成 Skill 使用文档

这份文档写给第一次使用的人。你不需要先懂 Remotion，也不需要会写代码，只要准备好口播音频和文稿，就可以让 Codex 按流程生成一条考研类知识讲解视频。

## 1. 这是什么

`kaoyan-remotion-video` 是一个 Codex Skill，用来把考研类口播内容做成知识讲解视频。

它不是一个固定模板，而是一套视频生产流程。它会先理解你的口播内容，再拆解视频结构、设计画面、安排动效、生成 Remotion 项目，最后先做 720p 预览，确认后再导出 4K。

适合这类视频：

- 在职考研政策讲解
- MBA / MPA / MEM / MPAcc 报考条件讲解
- 院校招生简章解读
- 考试科目调整提醒
- 择校择专业分析
- 备考规划和进度自查
- 高分学员经验讲解
- 考研热点资讯解释

不太适合这类视频：

- 真人出镜精剪
- Vlog、探校、采访类大量真实素材剪辑
- 高度剧情化短片
- 强广告特效片
- 没有文稿、没有字幕稿，只靠音频盲猜内容的高精度视频

## 2. 它能帮你做什么

这个 Skill 可以帮你完成：

- 根据口播音频和文稿拆分视频结构
- 判断内容里哪些地方需要事实核查
- 设计教育类知识视频的排版和配色
- 生成 Remotion 动画视频项目
- 接入口播音频
- 按口播节奏安排卡片、进度条、重点词动效
- 先导出 720p 预览
- 根据反馈返修画面和动效
- 通过后导出 4K 成片

一句话理解：  
它把“我有一段口播，想做成专业考研知识视频”这件事，拆成一套可复用的生产流程。

## 3. 使用前准备

最少准备两个文件：

```text
voice.mp3        口播音频
script.srt/txt   口播文稿，最好带时间轴
```

建议文件：

```text
source-note.md   政策、院校、招生简章等事实来源摘要
reference.mp4    你想参考的视频风格
reference.png    你想参考的排版截图
```

如果是政策类、院校类、报考条件类内容，请先确认事实来源。Skill 负责视频生成和表达优化，不负责凭空判断政策真假。

## 4. 怎么安装

把仓库里的这个文件夹复制到 Codex 的 skills 目录：

```text
kaoyan-remotion-video
```

Windows 常见路径是：

```text
C:\Users\<你的用户名>\.codex\skills\kaoyan-remotion-video
```

复制完成后，重启 Codex。

如果你是从 GitHub 下载，仓库地址是：

```text
https://github.com/hhdop/kaoyan-remotion-video-skill
```

下载后只需要安装里面的：

```text
kaoyan-remotion-video/
```

## 5. 最简单的使用方式

把音频和文稿发给 Codex，然后这样说：

```text
使用 $kaoyan-remotion-video，根据我的口播音频和文稿，生成一条横屏 16:9 的考研类知识讲解视频。先做 720p 预览，确认后导出 4K。
```

如果你不想让它直接开始做，可以先让它拆任务：

```text
使用 $kaoyan-remotion-video，根据这段口播音频和文稿，先不要直接渲染成片。
请先把任务拆成子任务：
1. 内容理解
2. 事实风险检查
3. 视频结构拆分
4. 视觉风格方案
5. 动效节奏方案
6. Remotion 实现计划
7. 720p 预览验收标准
8. 4K 导出标准

每一步都输出可检查结果，等我确认后再进入下一步。
```

这是推荐方式。它会慢一点，但质量更稳定，也更适合团队复用。

## 6. 推荐工作流程

### 第一步：内容理解

Codex 会先读你的音频、文稿和时间轴，判断：

- 主题是什么
- 受众是谁
- 核心信息是什么
- 视频属于政策解读、备考规划、经验分享，还是热点资讯

你要检查的是：它有没有理解错主题。

### 第二步：事实风险检查

如果内容涉及院校政策、招生简章、报考条件、考试科目、国家线、复试线、学费、学制等，Codex 会标出需要核对的事实。

你要检查的是：有没有官方来源，是否需要加“以官方最终公布为准”。

### 第三步：视频结构拆分

Codex 会把口播拆成几个视频段落，例如：

```text
开头钩子 → 背景说明 → 重点信息 → 案例展开 → 总结提醒
```

你要检查的是：这个结构是否符合你想表达的重点。

### 第四步：视觉风格方案

Codex 会给出配色、字体、卡片、进度条、标题和重点词方案。

默认方向是：

```text
专业、清楚、可信、年轻化的教育知识账号风格
```

你要检查的是：像不像你的账号，而不是像廉价 PPT。

### 第五步：动效节奏方案

Codex 会安排：

- 卡片怎么进出
- 重点词怎么强调
- 进度条怎么推进
- 每一段如何跟口播对齐

你要检查的是：动效是否服务理解，而不是为了炫技。

### 第六步：Remotion 实现

确认方案后，Codex 会开始建项目、写代码、接音频、生成视频。

一般会生成：

```text
src/
public/
out/
preview-frames/
```

你不需要手动改代码，只要看预览并提反馈。

### 第七步：720p 预览

先看 720p，不要一开始就导 4K。

重点检查：

- 音频是否完整
- 画面是否跟口播同步
- 有没有遮挡、溢出、闪烁
- 信息是否清楚
- 动效是否太呆板

### 第八步：4K 导出

720p 通过后，再让 Codex 导出 4K。

可以这样说：

```text
720p 预览通过，按当前版本导出 4K。
```

## 9. 新手最容易踩的坑

### 只给音频，不给文稿

不推荐。  
没有文稿时，Codex 只能先转写或猜测内容，字幕和画面结构更容易出错。

### 一上来就要求 4K

不推荐。  
4K 导出慢，发现问题后返修成本高。先看 720p 预览更稳。

### 政策内容不做事实核查

风险很高。  
涉及院校、专业、考试科目、分数线、学费、学制时，必须先确认官方来源。

### 参考视频只给截图

截图能看排版，但看不到动效。  
如果你想复刻动效节奏，最好给完整视频。

### 让画面信息太满

考研知识视频不是把 PPT 搬进视频。  
每一屏只突出一个重点，观众才看得懂。

## 10. 一句话总结

如果你是第一次用，就记住这句话：

```text
先给音频和文稿，让 Skill 拆任务；确认 720p 预览；最后再导出 4K。
```

这样最稳，也最适合团队长期复用。
