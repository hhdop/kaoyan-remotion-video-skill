# 考研知识视频 Skill 使用指南

`kaoyan-remotion-video` 用来把口播音频和带时间码的文稿制作成横屏考研知识讲解视频。你不需要先会 Remotion，也不需要提前手动配置所有工具；Skill 会先证明环境可用，再进入内容和画面制作。

这份指南只讲怎么使用。涉及院校、政策、考试科目等内容是否准确，由使用者自行确认；Skill 默认按照你提供的音频和 SRT 制作，不额外核查或改写事实。

当前支持 Windows 10/11 和横屏 16:9。首次联网可用于安装缺失的用户级运行环境，已有 Node、pnpm、Chrome/Edge 或 FFmpeg 时会优先复用。

## 1. 准备素材

最低需要：

```text
口播音频：MP3、WAV、M4A 或 AAC
时间文稿：SRT
```

推荐另外提供：

- 账号名称、固定角标和品牌偏好。
- 你喜欢的参考图片或参考视频。（第一次调试会比较耗时，之后可以复用）
- 是否需要完整逐字字幕。（因为这个会额外消耗token如果对字幕要求不高，也可以剪映生成）

音频是唯一时间基准。Skill 不会改写口播，也不会把尚未说到的结论提前高亮。

## 2. 安装 Skill

从仓库下载后，只安装这个目录：

```text
kaoyan-remotion-video/
```

放入：

```text
%USERPROFILE%\.codex\skills\kaoyan-remotion-video
```

重新启动 Codex。第一次制作时允许 Skill 联网补齐缺失工具即可。正常情况下不需要关闭安全中心。

## 3. 直接使用

把 MP3 和 SRT 发给 Codex，然后输入：

```text
使用 $kaoyan-remotion-video，根据我的口播音频和 SRT 文稿，
生成一条横屏 16:9 的考研知识讲解视频。
任务开始前先完成环境检查和内容类型判断。
音频是唯一时间基准，不要提前展示尚未口播的结论。
先打开 Remotion 浏览器预览并导出 720p，
我确认节奏和画面以后，再导出 4K。
```

如果想换风格，把参考图片或参考视频一起发给 Codex，并补充：

```text
参考素材只用于学习视觉风格和动效节奏。
请重点参考它的配色、版式、字体层级、转场速度和信息密度，
不要照抄原素材或文案。音频和 SRT 仍是内容与时间基准。
```

参考图片适合说明配色、排版、字体和整体质感；参考视频更适合说明转场、速度、停顿和动效丰富度。只提供其中一种也可以。

## 4. Skill 会怎样工作

1. 检查 Node、pnpm、浏览器、FFmpeg、项目文件、音频和 SRT。
2. 测量真实音频时长，生成连续时间轴。
3. 判断内容属于规划、资讯还是知识讲解。
4. 生成 Remotion 场景和浏览器预览。
5. 检查关键帧中的裁字、遮挡、过曝、提前露字和空占位框。
6. 导出并验证 720p。
7. 根据反馈返修。
8. 只有 720p 通过后才导出 4K。

三种内容类型的画面不会混用：

- 规划类：优先级、轨道、进度、复习闭环。
- 资讯类：来源、变化、影响、事实和行动。
- 知识类：问题、概念、例子、方法和总结。

## 5. 浏览器预览与 MP4

这两个结果不是一回事：

- Remotion Studio：实时预览，需要本地服务持续运行。
- 720p/4K MP4：已经写入磁盘的视频文件，不需要 Studio 继续运行。

Studio helper 会返回类似下面的真实地址：

```text
http://127.0.0.1:3010/Main
```

如果 3010 已占用，它会自动选择 3011、3012 等可用端口。不要看到旧地址打不开就反复重启；应使用本次返回的 URL，并确认状态为 `ready`。

在 Codex 桌面应用中，后台服务可能需要运行批准。受限终端可能在命令结束时回收后台进程，此时应批准持久启动或在本机终端以前台模式运行，而不是关闭 Windows 安全功能。

## 6. 为什么不需要先自己安装 Node

Skill 的 Windows bootstrap 会按顺序寻找：

1. 你显式指定的工具。
2. 系统 `PATH` 中已有的工具。
3. Codex 自带运行时。
4. Skill 的用户级运行时目录。

都找不到时，首次联网才会下载固定版本并验证校验值。因此可以自己安装 Node，但不是使用前的强制步骤。公司网络限制下载时，可安装 Node 24.x 后重新运行预检。

## 7. Windows 安全提示

不要关闭 Windows 安全中心或 Smart App Control。

如果 Remotion 能渲染 still，却在 MP4 阶段提示 bundled FFmpeg/compositor 被阻止，渲染 helper 会自动：

1. 渲染连续图片序列。
2. 验证没有缺帧或重复帧。
3. 使用可执行的外部 FFmpeg 合成 H.264 与口播音频。
4. 使用 FFprobe 验证分辨率、帧率、时长和音视频流。

失败时会保留帧目录和 `<视频>.render-report.json`，便于定位，不会只显示一句“渲染失败”。

## 8. 720p 验收清单

- 音频完整，结尾没有提前截断。
- 关键词和确认状态在口播落点后出现。
- 没有文字裁切、遮挡、闪烁或溢出。
- 每屏只强调一个主要信息。
- 章节切换不生硬，动效有进入、强调和收束。
- 黄色只是局部强调，没有大面积过曝。
- 右上角角标、标题和底部进度不会互相覆盖。
- 资讯看起来像资讯，规划看起来像规划。

确认后可以直接说：

```text
720p 预览通过，按当前版本导出并验证 4K。
```

## 9. 常见问题

### 每次为什么又检查环境

检查不是重复安装。它在确认当前项目、当前端口、当前素材和当前可执行文件仍然有效。浏览器服务、输出目录权限和素材路径会随项目变化。

### 为什么有 MP4 却打不开浏览器地址

MP4 渲染已经结束，但 Studio 服务没有运行。重新启动 Studio 并使用新返回的 `/Main` URL。

### 为什么画面像 PPT

通常是每段只做统一淡入、内容一次出现太多、没有持续动作和收尾。要求按参考视频分析主动作、次动作、持续动作和 settle，并降低单屏信息量。

### 为什么资讯类效果一般

轨道和进度表更适合规划类。资讯应优先表达来源、明确变化、受影响对象和行动，不要把所有内容做成计划看板。

### 想换成自己喜欢的风格怎么办

直接提供参考图片或参考视频，并说明最想保留的部分，例如“参考这个配色和排版”“学习这段视频的转场速度”“信息密度再低一些”。不需要自己分析 Remotion 代码。

### 需要 Codex 核查政策或院校信息吗

不需要。文稿事实由使用者自行确认。Codex 只根据已确认的音频和 SRT 做结构、同步、画面与渲染；如果你主动要求联网核查，再单独提出即可。

## 10. 最简使用文案

```text
使用 $kaoyan-remotion-video，根据我的口播音频和 SRT，
制作横屏 16:9 考研知识视频。文稿事实我已自行确认。
先检查环境并打开浏览器预览，导出 720p 给我确认，
确认后再导出 4K。参考图片或参考视频只用于学习风格。
```
